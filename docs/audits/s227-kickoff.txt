## Validator gate questions

1. **Endpoint shape:** What GSC API endpoint returns impressions/clicks/
   avg position for a site? Is it Search Analytics query API?
   What are the required parameters (siteUrl, dateRange, dimensions)?

2. **Data normalization:** What shape should `gsc_runs.data` take?
   Suggested: `{ impressions: N, clicks: N, avg_position: N, period_days: 30 }`.
   Is there a per-page breakdown needed or just site-level totals?

3. **Error handling:** Same as Zernio pattern — `status='error'` row with
   `api_error_code` + `api_error_msg`, HTTP 200 to caller.
